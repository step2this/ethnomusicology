A Open in new tab
System Native
Why This Option
The Helvetica lineage. SF Pro on Mac (what most DJs use), Segoe UI on Windows, Roboto on Android. Paired with system monospace for BPM. Zero bytes loaded. The closest you can get to "just Helvetica" on the web.
Works Well For
Maximum performance. Instant render. No FOUT ever. The purest tool aesthetic — the font is invisible, the data is everything. DJs on Mac get SF Pro, which is excellent at small sizes.
Watch Out For
Cross-platform inconsistency — the table looks different on Mac vs Windows vs Linux. Flutter Web's CanvasKit renderer may not always resolve to SF Pro on Mac. System monospace width varies by platform, which can affect BPM column sizing.
B Open in new tab
Inter Tight
Why This Option
Inter was designed for screens at 11-16px — exactly the sizes this product uses. Tall x-height, open apertures, built-in tabular numbers. Paired with JetBrains Mono for BPM data. 175KB one-time load for cross-platform consistency.
Works Well For
Dense data tables with small text (10-13px). The catalog looks identical on every OS. JetBrains Mono makes BPM numbers highly distinguishable. Inter's tabular figures align numbers even outside the mono column.
Watch Out For
175KB load on first visit. Inter is very common — if visual uniqueness matters, this won't help (but for a DJ tool, content is the differentiator). Slightly more geometric than Helvetica — clean but fractionally less warm.
C Open in new tab
Roboto Default
Why This Option
Flutter's built-in font. Zero load for the primary typeface. Roboto Mono matches Roboto's metrics perfectly (shared x-height, cap height, baseline). Material 3 components work out of the box with no TextTheme overrides.
Works Well For
Fastest Flutter implementation. No pubspec.yaml changes for the primary font. Roboto + Roboto Mono harmony is the best sans/mono family pairing. Only 75KB additional load (just the mono + Arabic).
Watch Out For
Looks like "a Material app." DJ software (Rekordbox, Serato, Traktor) uses system or custom fonts — none look like Google products. Missing weight 600. Roboto is less legible than Inter at 9-10px.
My Take
B (Inter Tight), but A is a legit alternative

The type scale is identical across all three — it's locked by the component design. The only decision is font family.


C (Roboto) is the easy path but pushes toward "Material app" rather than "DJ tool." Cut it.


A (System Native) is the Helvetica vibe you described loving. On your Mac it'll be SF Pro, which is beautiful. Zero load cost. The risk is Flutter Web's unpredictable system font resolution — you might not always get SF Pro through CanvasKit. And cross-platform consistency goes out the window.


B (Inter) is purpose-built for exactly the sizes this product uses (10-15px on screen). It gives you the clean, tight sans-serif aesthetic — Helvetica-adjacent but optimized for pixel rendering. Cross-platform consistency means your catalog looks the same everywhere. JetBrains Mono is the best mono for number-heavy data. The 175KB cost is a one-time investment.


If you're primarily developing and using this on Mac, A is great. If you want it to look the same for every DJ regardless of OS, go with B.
