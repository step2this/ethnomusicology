Chip Strip
Why This Option
Horizontal row of store pills mirrors the existing chip vocabulary (Camelot keys, source pills). Minimal visual disruption — just 32px of extra height. Chips scan left-to-right, matching the table's information flow. First-letter avatars + store names keep it readable without custom icons.
Works Well For
Desktop-first, density-obsessed DJ workflows. Users who want purchase links visible without overlays. Setlists where multiple tracks might be expanded simultaneously. The "DJ crate" aesthetic — everything is flat, inline, scannable.
Watch Out For
24px chip height misses the 44px touch target guideline for mobile. Four chips can crowd narrow viewports. The "B" = Beatport or Bandcamp ambiguity (mitigated by store name text). Layout shift pushes tracks below.
B
Open in new tab
Popover Tray
Why This Option
Zero layout shift — the track table stays perfectly stable. Floating popover with vertical store list gives maximum room for store names and excellent 48px touch targets. Follows the overlay pattern already in the locked component system (slide-over, sort menus). Standard Flutter PopupMenuButton implementation.
Works Well For
Dense setlists where vertical stability matters. Touch/tablet users who need large tap targets. Quick "tap → pick → done" interactions. Users who've already internalized dropdown/popover patterns from DJ software.
Watch Out For
Popover occludes tracks below the trigger. Needs click-outside dismiss logic and z-index management. Only one popover should be open at a time. Right-edge clipping on narrow viewports needs repositioning logic.

Inline Grid
Why This Option
Full-width 4-column grid gives each store maximum visual real estate and the largest touch targets (~150px x 44px per cell). Equal-width columns treat all stores equally. 1px gap dividers match the table's flat aesthetic. Grid creates the clearest visual break from the source attribution row above.
Works Well For
Tablet/touch users. Visually impaired users who benefit from large targets. Use cases where only 1-2 tracks are expanded at a time. Products that want purchase links to feel like a first-class feature, not an afterthought.
Watch Out For
Largest layout shift (+44px, nearly doubles a track's height). Grid pattern is foreign to the existing component system. Chevron trigger is less semantically clear than a shopping icon. Four columns need responsive collapse on narrow viewports.
My Take
Popover Tray (B) is the strongest fit

The zero-layout-shift property is the killer feature. In a dense DJ setlist at 40px rows, vertical stability matters — the "scan fast" promise breaks if expanding one track pushes everything below it. Options A and C both cause layout shift (32px and 44px respectively).

B also has the best touch targets (48px rows), the most room for store names (no truncation of "Juno Download"), and follows the popover pattern already in the locked component system. Implementation is straightforward — Flutter's PopupMenuButton covers most of the behavior.

The main trade-off is momentary occlusion of tracks below. But this is a quick interaction — tap, pick a store, new tab opens, done in under 2 seconds. The user is focused on the popover, not the tracks behind it.

If you prefer everything inline with zero overlays, Option A (Chip Strip) is the runner-up. It's the most compact and uses the existing chip vocabulary. Just needs mobile touch target adjustments.

Option C (Inline Grid) is the outlier — visually clear but structurally foreign to the component system, and the layout shift is the largest.
