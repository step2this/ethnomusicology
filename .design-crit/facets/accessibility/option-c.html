Proactive Additions (beyond audit scope)
High Contrast Mode AAA 1.4.6
Toolbar toggle: #000 background, #FFF text, 3px gold focus rings, brighter Camelot chips with border reinforcement. All text exceeds 7:1 contrast. Persisted in localStorage; respects OS high-contrast preference on first visit.
Reduced Motion Mode AAA 2.3.3
Respects prefers-reduced-motion automatically; manual toolbar override. Slide-over opens instantly, skeleton loaders show static placeholder, all transitions 0ms.
Screen Reader Track Descriptions
Each row has a comprehensive aria-label: "Fisher, Losing It, 124 BPM, key 6A compatible with 5A and 7A, high energy, Tech House, from Spotify." Includes harmonic mixing context (compatible keys).
Density Toggle (Studio / Comfortable)
Studio: 40px rows. Comfortable: 48px rows (Material 3's 48dp touch target minimum). Better for tablet use and motor impairments. All accessibility fixes apply in both modes.
Keyboard Command Palette
Ctrl+K / Cmd+K opens fuzzy-search overlay. Search tracks by name, apply sorts, filter by key/BPM. Power-user feature doubling as a fully keyboard-driven alternative to mouse interaction.
Focus-Visible Only
:focus-visible instead of :focus on all elements. Mouse users never see focus rings; keyboard users always see the 2px amber ring (3px gold in high contrast). Eliminates "why is there a ring" from mouse users.
Row position announcement (arrow key navigation)
↓ aria-live="polite": "Track 3 of 8. Four Tet, Baby."
Live region announces position and identity when navigating with arrow keys. Gives screen reader users the same spatial awareness sighted users get from scrollbar position and row highlighting.
Implementation Effort
Highest effort of three options -- honest assessment
Fixes (shared with Option B): ~3-4 days. Keyboard nav, focus trap, ARIA roles, live regions, skip link, alt text, energy labels, badge sizing, icon tooltips, scrim ESC.

Proactive additions: ~3-5 additional days. High Contrast needs a second color token set and theme-switching logic. Reduced Motion needs a motion preference service. Command palette is a standalone widget (input, fuzzy search, result list, keyboard nav). Density toggle requires row-height tokens and layout recalculation. Focus-visible needs cross-browser testing (Safari inconsistencies).

Total: ~6-9 days. Roughly 2x the effort of Option B. The proactive features are genuinely valuable but may be over-engineered for MVP. Consider: will the first 100 users need High Contrast mode? If not, ship Option B now and promote to C in a later sprint. The screen reader labels and focus-visible changes are low-cost and could be cherry-picked into B.
