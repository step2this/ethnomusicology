Centered Catalog
Why This Option
Classic web app. 12-column grid, 1200px max-width, centered. Enriched track tiles in a vertical list with generous margins. Clean, simple, ships immediately.
Works Well For
Fastest to build. Comfortable reading width. Trivially responsive to tablet. Modern SaaS feel.
Watch Out For
Doesn't feel like DJ software. Wastes space on widescreen. No obvious home for future browser pane. Track tiles are bulkier than the table rows DJs expect.
B Open in new tab
Full-Width Crate
Why This Option
DJ software density. Full-width track table with sortable columns (Title, Artist, BPM, Key, Genre, Source). 48px icon rail reserves space for future browser pane. The Rekordbox collection view on the web.
Works Well For
DJs who think in sortable columns. Maximizes visible tracks. Table naturally supports sort/filter in v2. Icon rail is a lightweight bridge to v2 browser pane.
Watch Out For
Table rows stretched on ultra-wide. Collapsed rail is a visible affordance for a non-existent feature. Table layout is functional-first, harder to make beautiful. Responsive degradation is complex.
C Open in new tab
Browser + Crate
Why This Option
Full Rekordbox layout from day one. Left browser pane (220px, resizable) with source filters in v1, growing to genres/artists/labels in v2. Main area is a full-width track table. The layout every DJ already knows.
Works Well For
Zero learning curve for any Rekordbox/Traktor/Serato user. Source filtering is immediately useful. Browser grows naturally over time. Resizable split lets DJs tune their workspace.
Watch Out For
"Coming in v2" placeholders feel half-baked. Browser pane only has 3 items in v1. Resizable split is more complex in Flutter. 220px panel for 3 filter items is a lot of dedicated space.
My Take
B or C — depends on whether source filtering earns the browser pane in v1

Cut A. A centered web app layout is wrong for a DJ tool. DJs expect density, full-width tables, and a workspace feel — not a SaaS content page.

C is the right destination — the Rekordbox browser+crate layout is what every DJ already knows. But in v1, the browser pane only holds source filters (All/Beatport/Spotify). That's 3 items in a 220px permanent panel. The "Coming in v2" placeholders telegraph an unfinished product.

B is the pragmatic v1. Full-width crate with a 48px icon rail that's ready to expand. No half-empty panels. When genres/artists/labels are ready, the rail unfolds into C's full browser. It's the same architecture, just with the browser collapsed until the content justifies it.

The deciding question: is "show me just my Beatport tracks" useful enough in v1 to justify a permanent browser pane? If yes, go C. If that filter can live in the toolbar as a dropdown, go B.
