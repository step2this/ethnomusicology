Design Identity — Ember Crate

Ember Crate is a warm, dense, professional DJ tool. Amber and copper tones on warm-tinted blacks evoke analog hardware — think Roland TR-808, not Spotify. The interface prioritizes scanning speed: flat five hierarchy, 40px dense rows, and keyboard shortcuts built for DJs who think in BPM and key. Every pixel serves the working DJ's set prep workflow.

Design Decisions
Defining the Product
Screen Inventory — Catalog-First option-c · 1 round user decided
1 primary screen (Track Catalog) + 1 secondary (Import Sheet overlay). Catalog is home base. DJs browse far more than they import — the catalog IS the product.
History
Options presented: A) Multi-Screen Hub, B) Import-First, C) Catalog-First
Decision: Immediate lock on C. Import is a drawer action, one of many future sources. Extensible architecture.
User signal: “DJs browse far more than they import. The catalog IS the product.”
Edge States — Context-Rich option-b · 1 round user decided
Tailored edge states per situation. Empty catalog shows source cards. URL errors show valid examples. Errors reassure “your tracks are safe.” Partial imports show per-track results.
History
Options presented: A) Minimal/Generic, B) Context-Rich
Decision: Immediate lock on B. Each error, empty, and loading state gets purpose-built messaging.
User signal: “Context-rich approach is the right fit for this product.”
Arranging the Pieces
Layout — Full-Width Crate option-b · 1 round user decided
Edge-to-edge track table with sortable columns. 48px collapsed icon rail reserves space for future browser pane. No max-width — DJ software density.
History
Options presented: A) Centered Card, B) Full-Width Crate, C) Split Panel
Decision: Immediate lock on B. V1 ships the dense track table; browser pane comes later when justified by content.
User signal: “V1 discipline — ship the dense track table now, expand later.”
Content Hierarchy — Flat Five option-c · 1 round user decided
5 co-primary fields (Artist, Title, BPM, Key, Energy) differentiated by visual treatment not size. Bold text, regular text, monospace, color chip, bar graph. 3 emphasis levels. Task-flexible scanning.
History
Options presented: A) Title-Led, B) BPM-Key-Led, C) Flat Five
Decision: Immediate lock on C. Rejects false hierarchies — all five data points matter equally during set prep.
User signal: “All five data points matter — artist, title, BPM, key, and energy. Don't pick a winner.”
Component Design — Crate + Board option-ac · 2 rounds user decided
14 components. C's structural patterns (slide-over, detail strip, multi-column sort, source pills) merged with A's compact design language (20px Camelot chips, 5px energy bars, 6px radius, 34px buttons).
History
Round 1: 3 options presented (A: Lean Crate, B: Studio Deck, C: Mixer Board). User cut A and B, kept C's structure but wanted A's compact design language. Requested A+C merge.
Round 2: A+C merge “Crate + Board” presented. Locked immediately.
User signal: “Merge, don't compromise. C's structural patterns are critical. A's compact language keeps it dense. Sortable columns are non-negotiable.”
Elevation & Shape — Wire Lift option-b · 1 round user decided
Flat table (no shadows). Shadows only on floating elements: slide-over, toasts, primary CTA. Scrim + shadow for overlay depth. 4-token radius scale (0/3/6/full).
History
Options presented: A) Shadow Deck, B) Wire Lift, C) Glass Layer
Decision: Immediate lock on B. Subtle differences between options — trusts designer recommendation on technical facets.
User signal: “Flat table + shadowed overlays makes sense.”
Making It Feel Right
Typography — Inter Tight option-b · 1 round user decided
Inter for UI + JetBrains Mono for BPM/data. 7-level discrete scale (15/13/13/12/11/10/9px). 3 weights (400/500/700). Tabular numbers. ~175KB one-time load.
History
Options presented: A) System Stack, B) Inter Tight, C) Mono-First
Decision: Immediate lock on B. Chose quality and cross-platform consistency over zero-load pragmatism.
User signal: “Type matters. Clean sans-serif, Helvetica vibes.”
Color System — Ember Crate option-d · 1 round user decided
Warm amber (#E8963A) primary, copper (#C06030) secondary on warm-tinted blacks (#0F0D0B). Warm white text. 24 warm-shifted Camelot key colors. Energy: teal to amber to copper-red. Roland TR-808 aesthetic.
History
4 options presented: A) Mondrian Board (light, primary colors), B) Indigo Crate (dark monochrome), C) Neon Deck (cyan/magenta neon), D) Ember Crate (warm amber/copper).
Decision: Instant connection with D. The warm analog palette clicked viscerally.
User signal: “It's like a Roland 808. Warm amber is the brand. Subtle hardware references are welcome.”
Density & Spacing — Studio option-b · 1 round user decided
40px rows, ~19 tracks visible in 900px viewport, 4px base unit. 32x32 art thumbnails. 7-token spacing scale (4/8/12/16/24/32/48px).
History
Options presented: A) Compact, B) Studio, C) Comfortable
Decision: Lock on B. Matches the wireframe density from all prior areas. Trusted designer recommendation.
User signal: “Not fussed about density — Studio matches the wireframes already approved.”
Final Check
Accessibility — Enhanced Access option-b · 1 round user decided
WCAG AA, 12/12 issues fixed. Keyboard row navigation, focus trap on slide-over, skip link, 2px amber focus rings, semantic table, live regions, shortcut hints, row count announcements.
History
Options presented: A) Standard AA, B) Enhanced Access, C) Power User
Decision: Immediate lock on B. The DJ audience actively seeks hotkeys and keyboard shortcuts — accessibility features double as power-user features.
User signal: “DJs actively seek hotkeys — keyboard shortcuts are a feature, not just compliance.”

User Voice

Design preferences discovered across 10 areas and 12 crit rounds.

  • Thinks like a working DJ, not a product manager — decisions rooted in set prep workflow
  • V1 discipline — ship what works, expand only when content justifies it
  • Rejects false hierarchies — all DJ data points matter equally during mixing
  • Appreciates merges over compromises — combine the best parts, never water down
  • Color is the differentiator — warm amber/copper is the brand identity
  • Roland TR-808 aesthetic — analog warmth, not digital cool
  • Keyboard access is a feature, not compliance — DJs actively seek hotkeys
  • Designs should feel “sick” — visceral, professional, dense
