# Spike: SP-009 Purchase Link Store Viability

## Hypothesis

Constructing search URL templates for major DJ music stores (Beatport, Traxsource, Juno Download, Bandcamp) will provide a viable "buy this track" experience. At least 2 stores should achieve >60% hit rate on a diverse 10-track test set to justify implementation.

## Timebox

- **Maximum Hours**: 4h
- **Start Date**: 2026-03-07
- **Status**: In Progress

## Questions to Answer

1. Can we construct working search URL templates for each store?
2. Do the stores block server-side fetches (bot protection)?
3. What is the hit rate per store across underground and mainstream DJ tracks?
4. Are affiliate programs available for revenue share?
5. Which stores should we prioritize for Phase 7?

## S1: URL Pattern Verification

### Beatport

- **URL Template**: `https://www.beatport.com/search?q={artist}+{title}`
- **Status**: WORKING
- **Test**: "Derrick May Strings of Life" returned multiple versions including Original Mix (96 BPM, E Minor), Rhythim Is Rhythim versions, remixes
- **Server-side fetch**: Works (200 OK, full HTML with results)
- **API**: Exists (v4 at api.beatport.com) but requires approval, non-commercial only. Not needed for search links.
- **Notes**: Best DJ metadata (BPM, key displayed in results). Search is generous with matching.

### Traxsource

- **URL Template**: `https://www.traxsource.com/search?term={artist}+{title}`
- **Status**: TEMPLATE ONLY (403 on server-side fetch)
- **Server-side fetch**: Blocked (403 with User-Agent header)
- **API**: Invite-only. "Public Metadata API v1" exists but keys are generated by invite only. Provider APIs (sales/release reporting) restricted to content providers.
- **Notes**: URL template works in browser. Cannot verify results server-side. For purchase links we just need the URL template — users click through in their browser.

### Juno Download

- **URL Template**: `https://www.junodownload.com/search/?q%5Ball%5D%5B0%5D={artist}+{title}`
- **Status**: TEMPLATE ONLY (403 on server-side fetch)
- **Server-side fetch**: Blocked (403 even with User-Agent header)
- **API**: None found
- **Notes**: URL pattern uses array-style query params (`q[all][0]=`). Works in browser. Juno is a major DJ download store with good underground catalog.

### Bandcamp

- **URL Template**: `https://bandcamp.com/search?q={artist}+{title}`
- **Status**: WORKING
- **Test**: "Strings of Life" returned 18 results including covers, remixes, and related tracks. Original Derrick May version not on Bandcamp (expected — classic releases often aren't).
- **Server-side fetch**: Works (200 OK)
- **API**: None (confirmed in MEMORY.md — already known)
- **Notes**: Best for independent/underground releases. Classic catalog is weak. Search works but results quality varies.

### Summary

| Store | URL Template | Server Fetch | API | Best For |
|-------|-------------|-------------|-----|----------|
| Beatport | `/search?q=` | Works | Gated | DJ tracks, has BPM/key |
| Traxsource | `/search?term=` | 403 | Invite-only | House, disco, soulful |
| Juno Download | `/search/?q[all][0]=` | 403 | None | Wide catalog, multi-format |
| Bandcamp | `/search?q=` | Works | None | Independent/underground |

**Key Finding**: Server-side verification is only possible for Beatport and Bandcamp. For Traxsource and Juno, we must use template URLs without confirming results exist. This is acceptable — the user clicks through to the store's search page and sees results (or not) in their browser. This is the standard pattern used by other music platforms.
